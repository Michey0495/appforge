## 議事内容

### 1. 前回TODOの進捗確認

前回の決定事項3件のうち、2件は完了済み。パフォーマンス改善の調査レポートが未着手。佐藤が今週中に着手する。

### 2. v2.3リリース後のフィードバック

リリースから2週間経過。ユーザーからのフィードバックを整理した。

ポジティブな意見:
- 検索機能の速度が改善されたとの声が多い
- 新しいフィルターUIが直感的でわかりやすい

改善要望:
- CSV出力のエンコーディングがShift_JISのみ。UTF-8にも対応してほしい
- ダッシュボードのグラフ表示期間をカスタマイズしたい

不具合報告:
- 特定条件で日付フィルターが正しく動作しない（チケット #1847）

### 3. v2.4の機能スコープ

v2.4に含める機能を議論し、以下の3機能に決定した。

- CSV出力のUTF-8対応（要望多数、工数小）
- ダッシュボードの表示期間カスタマイズ（要望多数、工数中）
- 日付フィルターの不具合修正（#1847）

API連携機能の追加はv2.5以降に先送りする。

### 4. QA体制の見直し

テスト自動化のカバレッジが低下傾向にある。現在58%で、目標の80%に届いていない。原因はフロントエンドのテストが不足していること。

鈴木からReact Testing Libraryの導入を提案。コンポーネント単位のテストを段階的に追加していく方針で合意した。

## 決定事項

1. v2.4のスコープをCSV UTF-8対応、ダッシュボードカスタマイズ、日付フィルター修正の3件に確定
2. API連携機能はv2.5に先送り
3. フロントエンドのテスト自動化にReact Testing Libraryを導入する
4. v2.4のリリース目標は4月第2週
